## 七章

ブランチ

### ブランチを使う理由

バージョンわけした時、例えば1系と2系があって、2系を開発しつつも、1系も保守して行きたいって時に

分岐させる。

* ブランチ名に/が使える。

知らんかった。

### ブランチのチェックアウト

git checkoutを使う。

checkout時、追跡されてないものはそのままになる。

しかし、うまくマージできない場合はエラーが発生する。

* 新しいブランチを作る場合は、git checkout -b <name>

* ブランチを削除する場合は git branch -d <name>
